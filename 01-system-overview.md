## 1. MES 是什么

MES（Manufacturing Execution System，制造执行系统）是工厂的**生产过程管家**。

在COROS，我们生产智能运动手表。一块手表从中框贴上SN码到最终装箱发货，中间要经过20多道工序。**MES的工作就是确保每一块手表都正确地走完了这些工序，且全程可追溯。**

### MES 做什么，不做什么

**✅ MES 负责：**
- SN 生成、绑定、解绑、状态管理
- 工序过站控制（扫码→校验→放行）
- 生产数据采集（测试结果、绑定记录）
- 工单管理（创建、执行、关单）
- 质量检测数据（FQC / OQC）
- 维修与作废处理
- 全流程追溯
- 生产数据整理供给下游（COROS后台）

**❌ 其他系统负责：**
- 生产计划排程 → ERP / APS
- 库存管理 → WMS
- 订单与渠道 → OMS
- 财务核算 → SAP

## 2. 系统组成

MES 由 **4个核心组件** 协同工作：

```
👤 使用者                    🖥️ 前端                 ⚙️ 后端                  💾 数据
管理人员（电脑） ────→ MES后台（Vue3）    ────→ NestJS 后端服务     ────→ PostgreSQL（8个Schema）
产线操作员（工位） ──→ Lowcoder 工位页面  ────→ n8n 工作流引擎      ────→ Redis（缓存）
车间大屏         ────→ 大屏看板           ────→ NestJS
产线设备         ────────────────────────→ n8n（扫码、测试数据）
```

### 各组件是干什么的

| 组件 | 定位 | 谁在用 |
|------|------|--------|
| **MES后台**（Vue3） | 管理页面：工单、SN查询、产品、工序、权限 | 管理人员、IE、线长 |
| **Lowcoder** | 产线工位页面：维修记录、SN生成、解绑 | 产线操作员 |
| **NestJS后端** | 后台API：登录鉴权、权限控制、数据查询、自动审计 | 被MES后台页面调用 |
| **n8n工作流** | 产线引擎：322个工作流，处理所有工序扫码绑定、校验、SN生成 | 被产线设备直接调用 |
| **PostgreSQL** | 核心数据库：SN、工单、生产记录、产品、工序，共8个Schema | 被NestJS和n8n共同读写 |

> **为什么有两个后端（NestJS 和 n8n）？**
> - **NestJS**：处理MES后台页面的请求，有完整的权限体系和审计日志
> - **n8n**：处理产线设备的实时请求，322个工作流对应各道工序，可视化编排方便修改
> - 两者共用同一个PostgreSQL数据库，各司其职

## 3. 与外部系统的关系

### 3.1 当前状态

```
🏭 产线设备 ──扫码绑定──→ n8n ──→ PostgreSQL
                                        ↓
                              凌晨汇总到 daily_info ──→ COROS后台
                              SN/包装数据 ──→ OMS
                              发货同步 ──→ 管易ERP
```

### 3.2 未来规划

```
SAP ──工单自动下达──→ NestJS ──→ PostgreSQL
WMS ──来料/退料────→ NestJS
                                        ↓
                              NestJS ──成品入库──→ WMS
                              PostgreSQL ──生产数据──→ OMS ──渠道数据补齐──→ COROS后台
```

> **MES的边界**：MES只负责到"生产完成 + 成品入库"。入库之后的仓储、渠道发货、订单履约都不属于MES。

| 外部系统 | 角色 | 对接内容 | 状态 |
|---------|------|---------|------|
| **产线设备** | 数据源 | 扫码绑定、测试数据 → n8n | ✅ 已对接 |
| **COROS后台** | 下游 | MES每日汇总生产数据(daily_info) → 后台查询 | ✅ 已对接 |
| **OMS** | 下游 | MES给生产数据，OMS统一管理渠道 | ✅ 已对接 |
| **管易ERP** | 下游 | 发货数据同步（未来归入OMS） | ✅ 当前在用 |
| **SAP** | 上游 | 工单自动下达 → MES | ⬜ 未来规划 |
| **WMS** | 上下游 | 上游：来料/退料；下游：成品入库 | ⬜ 未来规划 |

## 4. 核心概念速览

### SN（序列号）

**每个SN = 一块物理手表**，10位编码（如 `M1D26040001`）。SN有5种状态：

| status | 含义 | 能否操作 |
|--------|------|---------|
| 0 | 反投机 | ❌ |
| 1 | 已生成 | ✅ |
| 2 | 已绑定 | ✅ |
| 3 | 已失效（作废） | ❌ 终态 |
| 4 | 维修中 | ❌ 中间态 |

### 工单（JobOrder）

**一个工单 = 一批同型号产品的生产任务**。状态：已创建 → 生产中 → 已关闭。

### 工序

**产线上的每一步操作**。以W335机型为例有24道工序（点胶→绑定→测试→老化→包装）。每道工序对应一个n8n工作流。

## 5. 数据架构概览

MES使用**3种数据库、8个Schema**：

| Schema | 一句话定位 |
|--------|----------|
| **mes** | 生产核心：SN、工单、工序、绑定数据 |
| **mes_daily_data** | 下游数据供给：每日凌晨汇总给COROS后台 |
| **public** | 品质与售后：激活记录(243万)、不良记录、审批 |
| **sync** | SAP同步：工单缓存、SN物料映射 |
| **mes_jp** | 日本代工：日本工厂的生产数据 |
| **warehouseDelivery** | 发货：发货明细(73万)、订单、退款 |
| **YFConsumer** | 旧MES：SQL Server，历史绑定数据（只读） |
| **oms-service** | OMS：MongoDB，库存与Shopify |

## 6. 技术栈速查

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | Vue3 + Vite | 基于 RuoYi-Vue3 |
| 低代码 | Lowcoder | 产线工位页面 |
| 后端 | NestJS | carole-admin 框架 |
| ORM | Prisma | 数据库访问 + 自动审计 |
| 工作流 | n8n（自托管） | 322个产线工作流 |
| 队列 | BullMQ | 异步任务 |
| 数据库 | PostgreSQL 18 | 主数据库（8个Schema） |
| 缓存 | Redis | Token、分布式锁 |
| 设备通信 | MQTT | 产线设备实时通信 |
